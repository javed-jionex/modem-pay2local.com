Pay2local Introduction

We have a mobile banking method. Mobile banking has 3 different types: Agent, Merchant, and Personal. Right now we are serving 2 bank type support Bkash, Nagad.

Success:
{{ "description" | translate }}: The payment has been successfully approved.
Meaning: The transaction has been processed and approved without any issues.
Pending:
{{ "description" | translate }}: The payment is pending, awaiting approval from the Pay2local side.
Meaning: The transaction is in a temporary state, waiting for further processing or approval from the payment gateway (Pay2local).
Rejected:
{{ "description" | translate }}: The payment is rejected, and there should be a reason for the rejection.
Meaning: The transaction has been declined, and specific details about the rejection reason are provided. Merchants should review these details to understand why the payment was not approved.
Not Initialized:
{{ "description" | translate }}: The payment request is not initialized. The user did not select any bank method and did not enter the transaction ID in the case of a deposit.
Meaning: The payment process was not initiated properly. This could be due to the user not completing necessary steps, such as selecting a payment method or providing essential transaction information.
